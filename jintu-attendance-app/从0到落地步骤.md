# 轻量级考勤 App · 从 0 到落地步骤

> 在 **admin 项目同级或子目录** 新建本文件夹作为独立项目，admin 保持不动。  
> 产品：仅 iOS/Android，离线纯本地，只需班级名称 + 学生姓名。  
> **当前实现**：数据层已采用 **localStorage**（`src/store/storage.ts`），与文中「SQLite 可选」方案并存；若需从零按本文搭建，可先按阶段三用内存/localStorage 占位，再择机接 SQLite。

---

## 阶段一：项目初始化（约 15 分钟）

### 步骤 1.1 创建项目目录

在 `admin` 同级或你希望的位置创建新文件夹，例如：

```bash
# 若在 admin 所在目录下执行（即 jintu 或 未命名文件夹 2/jintu）
mkdir jintu-attendance-app
cd jintu-attendance-app
```

或直接在资源管理器中新建文件夹 `jintu-attendance-app`。

---

### 步骤 1.2 用 Vite 初始化 React + TypeScript 项目

在 `jintu-attendance-app` 目录下执行：

```bash
npm create vite@latest . -- --template react-ts
```

若提示目录非空，可先 `npm create vite@latest my-app -- --template react-ts` 再把 `my-app` 内所有文件（含隐藏）挪到 `jintu-attendance-app`，或选择覆盖。

安装依赖：

```bash
npm install
```

验证：`npm run dev` 能在浏览器打开默认 Vite 页面。

---

### 步骤 1.3 安装路由与样式

```bash
npm install react-router-dom
npm install -D tailwindcss @tailwindcss/vite
# 或 Tailwind 3：npm install -D tailwindcss postcss autoprefixer && npx tailwindcss init -p
```

- 若用 Tailwind 4：在 `vite.config.ts` 中加 `import '@tailwindcss/vite'`，并建 `src/index.css` 写 `@import "tailwindcss";`。
- 若用 Tailwind 3：`tailwind.config.js` 的 `content` 包含 `"./index.html", "./src/**/*.{js,ts,jsx,tsx}"`，`src/index.css` 写 `@tailwind base; components; utilities;`。

---

### 步骤 1.4 安装 Capacitor（应用壳）

```bash
npm install @capacitor/core @capacitor/cli
npx cap init "考勤" "com.jintu.attendance"
```

- 应用名与 Bundle ID 可按需修改；`webDir` 保持 `dist`（与 Vite 默认输出一致）。

先构建一次，再添加平台：

```bash
npm run build
npx cap add ios
npx cap add android
```

（iOS 需 macOS + Xcode；Android 需 Android Studio / SDK。未装可先跳过，后续再 `cap add`。）

---

### 步骤 1.5 安装 SQLite 与原生能力插件

```bash
npm install @capacitor-community/sqlite
npm install @capacitor/share @capacitor/filesystem @capacitor/status-bar
```

- `@capacitor-community/sqlite`：需与当前 Capacitor 主版本兼容（Cap 5/6 对应插件 5.x/6.x），不兼容时查该插件文档换版本。
- 若暂不接真机，可先不装 SQLite，用内存或 localStorage 占位，后续再接。

---

### 步骤 1.6 安装 UI 与业务依赖

```bash
npm install class-variance-authority clsx tailwind-merge
npm install @radix-ui/react-dialog @radix-ui/react-alert-dialog @radix-ui/react-popover @radix-ui/react-tooltip @radix-ui/react-slot
npm install lucide-react
npm install @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities
npm install date-fns react-day-picker
```

- 按需补充 Radix 组件（如 Button、Input、Label 可自写或从 shadcn 拷贝）。
- 若从 admin 拷贝 shadcn 组件，可只拷用到的（如 `button`, `dialog`, `input`, `popover`, `tooltip`）。

## 阶段二：工程配置（约 10 分钟）

### 步骤 2.1 配置 Vite：Hash 路由与别名

`vite.config.ts` 示例：

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  base: './',           // 相对路径，便于 Capacitor 加载
  resolve: {
    alias: { '@': path.resolve(__dirname, './src') },
  },
})
```

- `base: './'` 对 Hash 路由不是必须，但有利于本地打开 `dist/index.html` 时资源路径正确。

---

### 步骤 2.2 配置 TypeScript 路径

`tsconfig.json`（或 `tsconfig.app.json`）中确保：

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": { "@/*": ["./src/*"] }
  },
  "include": ["src"]
}
```

`vite.config.ts` 已用 `resolve.alias` 时，构建会按别名解析；路径 `@/store/...` 等可正常使用。

---

### 步骤 2.3 配置 Capacitor

`capacitor.config.ts` 示例：

```ts
import type { CapacitorConfig } from '@capacitor/cli'

const config: CapacitorConfig = {
  appId: 'com.jintu.attendance',
  appName: '考勤',
  webDir: 'dist',
  server: { androidScheme: 'https' },
}
export default config
```

- `webDir` 必须与 Vite 输出目录一致（默认 `dist`）。

---

### 步骤 2.4 创建目录结构

在 `src` 下创建空目录与占位文件，便于后续按文档实现：

```
src/
  store/         # db.ts, classes.ts, students.ts, attendance.ts, announcements.ts
  hooks/         # useClassList.ts, useClass.ts, useAttendance.ts, useAnnouncements.ts
  pages/         # ClassList.tsx, Attendance.tsx, History.tsx, Settings.tsx
  components/    # ui/, AttendanceStats, StudentAttendanceList, ...
  lib/           # period.ts, reportText.ts, safeArea.ts
  types/         # index.ts
  capabilities/  # share.ts, fileExport.ts, fileImport.ts（可选）
```

可先只建目录和 `types/index.ts`、`lib/period.ts` 等少量文件，避免空目录被 git 忽略时可加 `.gitkeep`。

## 阶段三：数据层（约 30 分钟）

### 步骤 3.1 定义类型 `src/types/index.ts`

```ts
export interface ClassEntity {
  id: string
  name: string
  studentOrder: string[]
  createdAt: string
  updatedAt: string
}

export interface StudentEntity {
  id: string
  name: string
  classId: string
  sortIndex: number
  createdAt: string
  updatedAt: string
}

export interface AttendanceSnapshot {
  id: string
  classId: string
  date: string       // YYYY-MM-DD
  period: number     // 0|1|2|3
  statusMap: Record<string, number>
  confirmedAt?: string
  updatedAt: string
}

export interface AnnouncementEntity {
  id: string
  classId: string
  content: string
  expirationType: 'today' | 'permanent' | 'custom'
  startsAt?: string
  expiresAt?: string
  createdAt: string
  updatedAt: string
}

// 点名页展示用
export interface Student {
  id: string
  name: string
  attendanceStatus: number  // 0 未到 1 已到 2 请假 3 晚到
}
```

---

### 步骤 3.2 SQLite 初始化与建表 `src/store/db.ts`

- 使用 `@capacitor-community/sqlite` 打开/创建库（如 `jintu_attendance.db`），执行建表 SQL。
- 建表语句与《点名页-最优技术方案（详细）》中「五、存储层详细设计」一致：`classes`（无 major_category）、`students`、`attendance_snapshots`、`announcements`。
- 暴露：`init()`（App 启动时调用一次）、`isInitialized()`，以及供下面 store 调用的执行接口（如 `run(sql, params?)`、`query(sql, params?)`，具体以插件 API 为准）。

若暂未接 SQLite，可先写内存版：用 `Map`/数组存四张表，接口与后续 SQLite 版一致，便于先跑通页面。

---

### 步骤 3.3 班级 CRUD `src/store/classes.ts`

- `getAll(): Promise<ClassEntity[]>`
- `getById(id): Promise<ClassEntity | null>`
- `insert(entity: ClassEntity): Promise<void>`
- `update(entity: ClassEntity): Promise<void>`
- `delete(id): Promise<void>`

`student_order` 存为 JSON 字符串，读写时 `JSON.parse`/`JSON.stringify`。

---

### 步骤 3.4 学生 CRUD `src/store/students.ts`

- `getByClassId(classId): Promise<StudentEntity[]>`
- `insert(entity): Promise<void>`（添加学生时业务层只传 `name` + `classId`，由 store 生成 id、sortIndex）
- `update(entity): Promise<void>`
- `delete(id): Promise<void>`
- `replaceListForClass(classId, students[])`（导入时整班替换，可选）

---

### 步骤 3.5 考勤快照 `src/store/attendance.ts`

- `get(classId, date, period): Promise<AttendanceSnapshot | null>`
- `upsert(snapshot): Promise<void>`
- `listByClassAndDate(classId, date?): Promise<AttendanceSnapshot[]>`
- 当前时段计算：`getCurrentPeriodId(): 0|1|2|3`（上午/下午/晚一/晚二）可放在 `lib/period.ts`，由本 store 或页面调用。

---

### 步骤 3.6 公告 CRUD `src/store/announcements.ts`

- `getByClassId(classId): Promise<AnnouncementEntity[]>`
- `insert` / `update` / `delete`，按需过滤 `starts_at`/`expires_at` 在业务层做。

---

## 阶段四：工具与 Hooks（约 20 分钟）

### 步骤 4.1 时段计算 `src/lib/period.ts`

根据本地时间返回 0（上午）/1（下午）/2（晚一）/3（晚二），逻辑与 admin 的 `getCurrentPeriodId` 一致（如 5–12 上午，12–18 下午，18:00–19:30 晚一，其余晚二）。

---

### 步骤 4.2 报告文案 `src/lib/reportText.ts`

入参：班级名、日期时段字符串、学生列表（含 attendanceStatus）。  
返回：轻量版报告文案（仅班级名、日期时段、应到/实到/请假/晚到/未到人数与名单），不含大类、教学老师到岗。

---

### 步骤 4.3 Hooks（按页需要逐步加）

- `useClassList()`：从 store 读班级列表，返回列表 + 新增/删除方法。
- `useClass(id)`：读单个班级 + 其学生 + 公告 + 当前时段快照，返回合并后的数据与更新方法。
- `useAttendance(classId)`：读/写当前 (classId, date, period) 快照，封装状态切换、全员到齐、重置。
- `useAnnouncements(classId)`：读/写该班公告。

---

## 阶段五：路由与页面（约 1–2 小时）

### 步骤 5.1 路由入口 `src/App.tsx`

- 使用 `HashRouter`、`Routes`、`Route`。
- 路由表：`/` → ClassList，`/attendance/:classId` → Attendance，`/history/:classId?` → History，`/settings` → Settings。
- `main.tsx` 只渲染 `<App />`，`App` 内包 `HashRouter` 与 `Routes`。

---

### 步骤 5.2 班级列表页 `src/pages/ClassList.tsx`

- 调用 `useClassList()` 展示班级列表；每项点击 `navigate(\`/attendance/${id}\`)`（即选择班级）。
- 顶部或浮动按钮「新增班级」：弹窗或简单页，**仅填班级名称**，提交调 `store.classes.insert`，再刷新列表或跳转该班。
- 入口「设置」→ `navigate('/settings')`；可选「导入」入口（见阶段六）。

---

### 步骤 5.3 点名页 `src/pages/Attendance.tsx`

- 从 `useParams()` 取 `classId`，用 `useClass(classId)` 拿班级、学生、公告、当前快照；无快照则全部未到。
- 学生列表：展示姓名与状态（未到/已到/请假/晚到），点击切换状态并立即写回 `store.attendance`。
- 拖拽排序：更新顺序后写回 `store.classes` 的 `student_order` 与 `store.students` 的 sortIndex。
- 「添加学生」：弹窗或页内表单，**仅填姓名**，提交生成 id/sortIndex 并调 `store.students.insert`，并更新班级的 `student_order`。
- 搜索：前端按姓名过滤当前列表。
- 公告、考勤报告（预览/复制/分享/确认）、全员到齐、重置、时段自动重置：逻辑与《点名页-离线纯本地设计方案》一致；报告确认只写快照的 `confirmedAt`。
- 导航栏：返回（回列表）、更多（全员到齐、重置、历史入口等）。

可从 admin 的 `app/attendance/[id]/page.tsx` 及子组件迁入，删除所有 `fetch`/Socket，改为上述 store 与 hooks。

---

### 步骤 5.4 本地历史页 `src/pages/History.tsx`

- 按班级（可选）、日期筛选，从 `store.attendance` 的 `listByClassAndDate` 等取已确认快照列表。
- 详情展示某条快照的应到/实到/请假/晚到名单。
- 导出：生成 JSON/CSV 后通过 Share 或 Filesystem 保存（见阶段六）。

---

### 步骤 5.5 设置页 `src/pages/Settings.tsx`

- **导入**：从系统选文件（或剪贴板粘贴 JSON），解析后写入 `store.classes` + `store.students`；格式与《最优技术方案》中导出 JSON 一致（班级无 majorCategory）。
- **导出**：从 store 读出班级+学生（可选快照/公告），生成 JSON，调 Share 或保存到文件。
- 可选「关于」、清空数据等。

---

## 阶段六：原生能力（约 20 分钟）

### 步骤 6.1 分享报告

- 点名页报告预览后，提供「复制」与「分享」。
- 分享：`import { Share } from '@capacitor/share'`，`Share.share({ title: '考勤报告', text: reportText })`。

---

### 步骤 6.2 导出文件并分享/保存

- 将导出 JSON 用 `Filesystem.writeFile` 写到临时目录（Capacitor 的 cache/documents），得到 `path` 后 `Share.share({ url: path })`，用户可选「保存到文件」或分享到其他 App。

---

### 步骤 6.3 导入：选择文件

- 使用社区插件（如 `capawesome/capacitor-file-picker`）或原生桥接，让用户选择文件；读取内容后解析 JSON 并写入 store。
- 若无选文件能力，可先做「从剪贴板粘贴 JSON」导入。

---

### 步骤 6.4 安全区与状态栏

- 根布局或主内容区加 `padding: env(safe-area-inset-top) ... env(safe-area-inset-bottom)`；底部栏用 `pb-safe` 等避免被 Home 条遮挡。
- `StatusBar.setStyle({ style: 'DARK' })` 等与主题一致（可选）。

---

## 阶段七：Capacitor 与真机运行（约 15 分钟）

### 步骤 7.1 同步与运行

## 附录：与 admin 的对照

| 项目 | admin（现有） | jintu-attendance-app（本项） |
|------|----------------|------------------------------|
| 数据来源 | API + Socket | 本地 SQLite + 导入 |
| 班级字段 | 名称 + 大类 | 仅名称 |
| 学生字段 | 姓名等 | 仅姓名（id/排序系统生成） |
| 报告内容 | 含大类、老师到岗 | 仅班级名、时段、人数与名单 |
| 路由 | Next.js App Router | HashRouter，仅 4 个页面 |

本步骤文档与《点名页-离线纯本地设计方案》《点名页-最优技术方案（详细）》一致；实现时以两篇方案文档为准，本文仅做从 0 到落地的顺序与拆分说明。
