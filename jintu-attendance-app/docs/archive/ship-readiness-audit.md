# 可落地性审查与整改蓝图

> 该文档已从正式维护入口移入归档，保留为阶段性审查摘要。

## 摘要结论

这份审查的核心判断是：项目已经具备完整产品骨架，但仍更接近“可演示原型”，还不是“可稳定交付老师使用的工具”。

当时收敛出的 4 组核心问题是：

1. 数据真值不可靠
2. 关键入口和产品心智分裂
3. 高风险自动化过强
4. UI / 排版 / 信息架构偏展示稿

## 建议优先级

后续整改顺序应固定为：

1. 数据真值
2. 关键入口可达
3. 交互闭环
4. UI 视觉统一

## 对标判断摘要

对标结论是：当前项目更接近 `TeacherKit`、`Teacher Aide Pro` 这类个人教师工具，而不是家校协同或学校平台。

更值得补的方向包括：

- 学生资料卡与异常学生识别
- 更适合课堂现场的名单 / 视图切换
- 趋势型统计和更完整的报告输出
- 设备备份、恢复与迁移

不建议在当前阶段优先投入：

- 家校沟通
- 多角色权限
- 学校后台
- 复杂 SaaS 能力
