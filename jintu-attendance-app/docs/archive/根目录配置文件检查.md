## 二、交叉引用一致性

| 项目 | 说明 |
|------|------|
| **路径别名** | tsconfig.json `paths` 与 vite.config.ts `resolve.alias` 均为 `@` → `./src`，一致。 |
| **输出目录** | Vite 默认输出 `dist`，capacitor.config.ts `webDir: 'dist'`，一致。 |
| **环境变量** | .env.example 与 src/vite-env.d.ts、src/lib/appConfig.ts 中使用的变量一致。 |
| **Tailwind** | content 覆盖 index.html 与 src 下所有 .ts/.tsx；tailwindcss-animate 在 devDependencies，已用 ESM import 引入。 |

## 三、已做修改

- **tailwind.config.js**：在 `"type": "module"` 下将 `require('tailwindcss-animate')` 改为 `import tailwindcssAnimate from 'tailwindcss-animate'`，避免在严格 ESM 环境下报错，并与根目录 ESM 一致。

## 四、可选说明

- **index.html 的 title**：当前为静态「稍微一点」；运行时由 main.tsx 通过 `document.title = getAppName()` 覆盖，无问题。若希望构建时即注入 VITE_APP_NAME，需使用 Vite HTML 插件或占位替换。
- **eslint 忽略 *.config.js/ts**：配置类文件未参与 lint，属常见做法；若需对 vite/tailwind 等配置做检查，可单独放宽 ignores 或为 config 文件开规则。
