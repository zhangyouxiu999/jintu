# 架构说明

## 技术栈

- `Vite`
- `React 18`
- `TypeScript`
- `Tailwind CSS`
- `Capacitor 6`

## 整体结构

项目采用 Web 前端 + Capacitor 原生壳的方式交付：

- 前端代码负责页面、状态与本地数据交互
- Capacitor 提供 Android / iOS 容器与原生能力接入
- 构建产物输出到 `dist/`，再同步到 `android/` 与 `ios/` 工程

## 路由

项目基于 `HashRouter`，适合 Capacitor 场景下的本地资源加载方式，可降低原生容器内路由刷新的适配复杂度。

## 数据存储

当前以本地存储为主，核心约定：

- 数据默认保存在设备本地
- 更新版本时应尽量保持存储键与结构兼容
- 若后续引入迁移逻辑，建议增加数据版本号

## 原生能力

项目已接入或围绕 Capacitor 设计以下能力：

- 分享
- 文件导出
- 剪贴板
- 生命周期与返回键
- 安全区适配

相关改动应优先遵循 Capacitor 官方能力与 Web 回退并存的思路。

## Android 兼容性要点

- 使用原生返回键事件处理 Android 返回行为
- 原生平台下优先通过 Capacitor 剪贴板能力写入内容
- 文件导出优先走“生成文件 -> 分享”的移动端方案
- 页面布局考虑 `safe-area-inset-*`
- 当前构建参数基于 Capacitor Android 工程维护

## 设计边界

本项目聚焦独立移动端应用，不依赖上一级目录中的管理端工程。文档、构建流程与原生工程约定均以当前目录为准。
