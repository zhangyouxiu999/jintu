# 产品与交互说明

## 文档定位

- 产品名称：考勤 App
- 版本基线：`2026-04-02`
- 文档性质：当前版本唯一的产品真值文档

本文档合并了原先的设计原则、PRD 和原型说明，统一回答这 3 类问题：

- 这个产品是什么，不是什么
- 用户在 App 里应该怎么走主链路
- 每个页面和交互在当前版本里分别承担什么职责

## 1. 产品定位

这是一个老师个人使用的移动端点名主场工具，不是家校协同平台，也不是多角色教务系统。

产品心智固定为：

`我 -> 当前班级 -> 点名 -> 当前班级扩展能力`

当前版本强调以下体验：

- 打开即回到当前班级点名页
- 所有能力都围绕“当前班级”展开
- 低频工具与高频课堂动作明确分层
- 默认本地优先，不依赖复杂后端部署

## 2. 目标与非目标

### 2.1 目标

- 支持班级级别的日常点名和状态记录
- 支持已确认考勤历史的查看与导出
- 支持每个班级维护学生名单、课表和成绩
- 支持通过模板进行导入导出
- 支持移动端单人、高频、低学习成本的使用方式

### 2.2 非目标

- 不建设复杂云同步或多人协同
- 不提供多角色权限体系
- 不承担家校沟通或消息平台能力
- 不建设完整 SaaS 管理后台

## 3. 核心原则

### 3.1 点名页就是首页

- 登录后直接进入当前班级点名页
- 不设置底部多模块一级导航
- 用户不需要先理解模块结构，再开始完成点名

### 3.2 一切能力从当前班级展开

- 学生名单、课表、成绩、历史都属于“这个班”的扩展能力
- 这些能力必须从当前班级面板进入，而不是与点名并列争抢首页入口

### 3.3 当前班级必须持续可感知

- 顶部持续展示当前班级
- 顶部班级入口同时承担“切换班级”和“打开班级扩展”的职责

### 3.4 点名页只保留课堂内动作

- 保留：点名、公告、一键全勤、重置考勤、生成报告
- 拆出：学生增删改排序、批量导入导出

### 3.5 更多页和班级管理页分层承载低频能力

- 更多页承载当前班级切换、模板下载、自动重置考勤、退出登录等低频动作
- 班级管理页负责新增、改名、删除、整班导出等完整班级维护

## 4. 目标用户

- 班主任：高频点名、查看历史、维护班级信息
- 任课老师：快速完成某节课点名和公告提醒
- 教务辅助人员：批量维护名单、课表、成绩和导出表格

## 5. 信息架构

```text
登录
  ├─ 无班级
  │   └─ 首次建班流程
  │       └─ 当前班级点名页
  └─ 有班级
      └─ 当前班级点名页

当前班级点名页（首页）
  ├─ 当前班级面板
  │   ├─ 我的班级（切换 / 添加 / 班级管理）
  │   └─ 当前班级扩展（学生名单 / 课表 / 成绩 / 历史）
  ├─ 课堂内动作（公告 / 一键全勤 / 重置 / 报告）
  └─ 更多工具页（模板 / 偏好 / 退出登录）
```

## 6. 核心使用链路

### 6.1 首次使用

- 用户登录
- 如果没有班级，进入首次建班流程
- 可选导入第一批学生
- 完成后进入该班点名页

### 6.2 日常点名

- 打开 App
- 自动进入当前班级点名页
- 对学生标记已到、请假、晚到、未到
- 必要时发布公告、一键全勤、重置考勤、生成报告

### 6.3 当前班级扩展

- 通过顶部班级面板进入学生名单、课表、成绩、历史
- 所有扩展页面都默认带当前班级上下文

### 6.4 低频工具

- 通过更多页下载模板
- 配置自动重置考勤
- 退出登录

## 7. 页面职责

### 7.1 登录页

- 品牌区
- 账号密码表单
- 登录按钮
- 登录成功后直接进入首页主链路

### 7.2 首次建班流程页

- 步骤提示
- 班级命名
- 可选学生导入
- 完成后进入该班点名页

### 7.3 点名页

- 作为默认首页
- 展示当前班级
- 展示学生出勤状态
- 支持状态筛选
- 支持一键全勤、重置、公告、报告
- 不承载学生名单维护职责

当班级没有学生时，空状态必须明确引导用户进入学生名单页。

### 7.4 学生名单页

- 当前班级上下文与人数摘要
- 可滚动名单列表
- 支持拖拽排序和序号编辑
- 编辑、删除在名单维护页完成
- 添加、导入、导出属于该页职责

### 7.5 课表 / 成绩 / 历史页

- 继续保留独立页面
- 入口全部下沉到当前班级面板
- 页面内部默认在当前班级上下文中工作

### 7.6 更多页

- 当前班级切换
- 跳转到班级管理页
- 模板下载
- 自动重置考勤开关
- 退出登录

### 7.7 班级管理页

- 班级列表
- 编辑、删除、整班导出
- 新增班级
- 作为低频深层页面存在

## 8. 功能范围

### 8.1 登录与首次建班

- 基础账号密码登录
- 无班级时进入首次建班流程
- 建班后可选导入学生名单

### 8.2 点名页

- 作为默认首页
- 展示当前班级
- 展示学生出勤状态
- 支持状态筛选
- 支持一键全勤、重置、公告、报告

### 8.3 学生名单页

- 新增学生
- 批量导入名单
- 导出学生名单
- 编辑学生姓名
- 删除学生
- 调整学生顺序

### 8.4 当前班级扩展页

- 课表页：查看、编辑、导入、导出
- 成绩页：多期成绩管理、导入、导出
- 历史页：查看已确认考勤、导出

### 8.5 更多页

- 当前班级切换
- 跳转到班级管理页
- 模板下载
- 自动重置考勤开关
- 退出登录

### 8.6 班级管理页

- 新增班级
- 修改班级名称
- 删除班级
- 导出整班数据

## 9. 路由基线

- `/login`
- `/class-setup`
- `/`
- `/attendance/:classId`
- `/students/:classId`
- `/schedule/:classId`
- `/grades/:classId`
- `/history/:classId`
- `/more`
- `/classes`

兼容约定：

- `/schedule`、`/grades`、`/history` 自动重定向到当前班级版本
- `/settings`、`/templates` 自动重定向到 `/more`

## 10. 状态规范

每个页面至少覆盖：

- 加载中
- 正常态
- 空状态
- 危险操作确认态
- 操作完成反馈态

其中点名页和学生名单页优先保证：

- 无班级
- 有班级但无学生
- 多班级切换
- 导入导出
- 重置与删除确认

## 11. 验收要点

- 登录后有班级时直接进入当前班级点名页
- 登录后无班级时进入首次建班流程
- 点名页不再承担学生名单维护职责
- 顶部班级面板可切换班级并进入学生/课表/成绩/历史
- 更多页保留低频工具与轻量班级入口，但不承载完整班级维护闭环
- 旧路由仍能安全跳转到新入口

## 12. 明确不做

- 不做底部多模块一级导航
- 不做家长端、家校沟通、消息协同
- 不做多角色、多端统一管理后台
- 不做云端多人同时编辑

## 13. 商用与交付边界

### 13.1 当前更适合的交付方式

当前项目更适合“售卖账号 + 本地 App”的交付方式，即用户通过安装包使用应用，业务数据主要保存在本机。

### 13.2 两种交付模式

#### 方式一：一客户一包

特点：

- 无需后端
- 为每个客户单独构建安装包
- 可在构建时写入专属账号或品牌信息

适用场景：

- 客户数量可控
- 偏定制化交付
- 不依赖统一应用商店分发

#### 方式二：通用包 + 多账号

特点：

- 只维护一个通用安装包
- 用户通过账号密码登录
- 需要一个最小登录校验服务

适用场景：

- 希望统一分发
- 需要后续增删账号、设置有效期
- 有一定运维能力

### 13.3 风险与控制

若担心 APK 转发或一号多用，可优先考虑方式二，并增加：

- 设备绑定
- 定期联网校验
- 账号状态管理

若使用方式一，则更偏向商务约束与人工交付管理，技术层面很难完全防止转发。

### 13.4 推荐实施顺序

1. 先确认采用方式一还是方式二
2. 方式一优先完善构建与交付流程
3. 方式二再补最小认证服务与账号管理
4. 最后补充隐私政策、发布规范与更新策略

### 13.5 商用注意事项

- 生产环境不要使用默认账号
- 品牌名称与登录凭据建议通过环境变量注入
- 若需要长期演进，尽量提前规划版本、签名、账号与更新策略
